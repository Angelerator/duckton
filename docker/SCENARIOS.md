# Off-chain scenario catalog — Duckton grid

This file is the written scenario list AND the spec the Docker harness covers. It
is reproduced verbatim from the harness brief. The harness in `docker/` maps each
catalog id to either a live-swarm assertion (real QUIC across containers, exact
rows/log lines/error substrings) or a library-tier determinism check (`cargo test`
against the `MockEngine`/in-memory rails) for the adversarial/internal cases the
live extension cannot inject. See `docker/REPORT.md` for the id→evidence mapping.
